= Listar todas las notificaciones
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/notifications`#

Devuelve una matriz JSON de todos los recursos coincidentes de la colección. Si se especifican campos individuales en la solicitud, la matriz JSON de elementos contendrá los campos solicitados para cada recurso coincidente en el orden especificado.



== Parámetros

|===
| Nombre | Tipo | En | Obligatorio | Descripción 


| account_id | cadena | ruta | Verdadero  a| 
ID del recurso de cuenta que lo contiene

* ejemplo: {{.Account}}




| incluya | cadena | consulta | Falso  a| 
Indica qué campos deben devolverse al listar una colección.

Ejemplos de URL: "include=id", "include=id,name"

Ejemplos OpenAPI: "id"

Ejemplos de URL: "limit=2"

Ejemplos OpenAPI: "2"

* ejemplo: 4

Indica qué campos deben coincidir para devolver un recurso al listar una colección.

Los operadores admitidos son:

* lte/gte - Menor o igual que/Mayor o igual que


Ejemplo de URL: "filter=field%20eq%20%27value%27"

Ejemplos OpenAPI: "field eq 'value'"

Ejemplos de URL: "skip=1"

[role="api-collapsible-fifth-title"]
data

Objeto JSON que contiene información adicional específica del evento.

[cols="4*"]
|===
| Nombre | Tipo | Obligatorio | Descripción 


| ttl | número | Falso  a| 
Número JSON que indica el número de segundos (desde eventTime) que se conservará el evento. Este campo solo se incluye cuando un evento se elimina automáticamente después del número de segundos especificado. Cuando se elimina, si el evento tiene un destino de notificación, también se eliminan todas las notificaciones asociadas. Cuando se elimina, si el evento tiene un destino de banner, también se elimina el banner. La ausencia de este campo (o un valor de "0") significa que un evento no se eliminará automáticamente.



| isAcknowledgeable | cadena | Falso  a| 
Cadena JSON que indica si el banner se puede descartar. Los valores definidos son:

* "true" - La pancarta se puede descartar
* "false" - La pancarta no se puede descartar

notification_1.3_get_response_body

[cols="4*"]
|===
| Nombre | Tipo | Obligatorio | Descripción 


| tipo | cadena | Verdadero  a| 
Tipo de medio del recurso. Los valores definidos son:

* "application/astra-notification"




| versión | cadena | Verdadero  a| 
Versión del recurso. Los valores definidos son:

* "1,0"
* "1,1"
* "1,2"
* "1,3"




| id | cadena | Verdadero  a| 
Identificador único global del recurso. Los valores definidos son:

* Cumple con el esquema UUIDv4




| nombre | cadena | Verdadero  a| 
Cadena JSON que contiene un nombre jerárquico en notación de puntos (.) para el evento. Los valores definidos son:

* Longitud mínima: 3
* Longitud máxima: 127
* La expresión regular definida es: "^(([a-z])_(\.))_([a-z])+$"




| sequenceCount | número | Verdadero  a| 
Número JSON que contiene un recuento de secuencias. El número es incrementado por el sistema cada vez que se recibe un nuevo evento.



| resumen | cadena | Verdadero  a| 
Cadena JSON que contiene el resumen abreviado (menos de 40 caracteres) del evento. Los valores definidos son:

* Longitud mínima: 3
* Longitud máxima: 79




| eventTime | cadena | Verdadero  a| 
Cadena JSON que contiene una marca de tiempo que indica cuándo se produjo un evento. Los valores definidos son:

* Cumple con el esquema de fecha y hora ISO-8601




| fuente | cadena | Verdadero  a| 
Cadena JSON que contiene el nombre en minúsculas del servicio que genera un evento. Los valores definidos son:

* Longitud mínima: 1
* Longitud máxima: 19
* La expresión regular definida es: "^([a-z-])*$"




| resourceID | cadena | Verdadero  a| 
Cadena JSON que contiene el ID del recurso asociado a un evento. Los valores definidos son:

* Cumple con el esquema del identificador Astra




| additionalResourceIDs | array[string] | Verdadero  a| 
Matriz JSON de cadenas JSON, cada una de las cuales contiene un ID de recurso de un recurso adicional asociado al evento. Aunque el campo "resourceID" contiene el recurso principal asociado a este evento, este campo contiene cualquier número de ID de recursos adicionales que se asocien a los mismos eventos. Si no hay recursos adicionales asociados al evento, la matriz estará vacía. Los valores definidos son:

* Cumple con el esquema del identificador Astra




| resourceType | cadena | Verdadero  a| 
Cadena JSON que contiene el tipo de medio del recurso asociado a un evento. Los valores definidos son:

* Longitud mínima: 4
* Longitud máxima: 79
* La expresión regular definida es: "^application\/astra-([a-zA-Z])+$"




| correlationID | cadena | Verdadero  a| 
Cadena JSON que contiene un ID de correlación UUIDv4 que permite agrupar eventos relacionados. Los valores definidos son:

* Cumple con el esquema del identificador Astra




| gravedad | cadena | Verdadero  a| 
Cadena JSON que contiene uno de los siguientes valores que indican la gravedad de un evento. Los valores definidos son:

* "cleared" - Indica que un evento previamente reportado que describe una condición ya no está activo. Un evento con gravedad "cleared" usará el mismo resumen e ID de correlación para permitir su cotejo.
* "indeterminado" - Indica que no se puede determinar la gravedad.
* "informativo" - Indica un cambio de estado que es esperado y normal (no definido en X.733).
* "alerta" - Indica la detección de un fallo potencial o inminente que afecta al servicio, antes de que se hayan producido efectos significativos.
* "critical" - Indica que se ha producido una condición que afecta al servicio y se requiere una acción correctiva inmediata. Las definiciones de "cleared", "indeterminate", "warning" y "critical" se basan en las definiciones de la cláusula 8.1.2.3 de CCITT X.733.




| clase | cadena | Verdadero  a| 
Cadena JSON que contiene uno de los siguientes valores que indican la clase del evento. Los valores definidos son:

* "sistema" - El evento está relacionado con un evento o solicitud interna.
* "usuario" - El evento está relacionado con un evento o solicitud iniciado por el usuario.
* "seguridad" - El evento está relacionado con un problema de seguridad.




| descripción | cadena | Verdadero  a| 
Cadena JSON que contiene un texto descriptivo largo sobre el evento. Los valores definidos son:

* Longitud mínima: 3
* Longitud máxima: 1023




| descripciónURL | cadena | Falso  a| 
Cadena JSON que contiene la URL que habilita y enlaza un botón del banner a documentación externa. Los valores definidos son:

* Longitud mínima: 3
* Longitud máxima: 4095




| correctiveAction | cadena | Falso  a| 
Cadena JSON que contiene un texto descriptivo largo sobre la acción correctiva que debe tomarse en respuesta al evento. Los valores definidos son:

* Longitud mínima: 3
* Longitud máxima: 1023




| correctiveActionURL | cadena | Falso  a| 
Cadena JSON que contiene la URL que vincula un botón del banner a una página de la interfaz de usuario para solucionar el problema. Los valores definidos son:

* Longitud mínima: 3
* Longitud máxima: 4095




| visibilidad | array[string] | Falso  a| 
Matriz JSON de cadenas JSON, cada una de las cuales indica un rol RBAC permitido para ver el evento. Los roles se definen juntos con OR. Opcional. Si se omite, un usuario con cualquier rol RBAC puede ver el evento. Los valores definidos son:

* Longitud mínima: 1
* Longitud máxima: 63




| destinos | array[string] | Falso  a| 
Matriz JSON de cadenas JSON, cada una de las cuales indica un destino adicional para el evento. Permiten a un servicio proporcionar pistas sobre dónde debe enrutarse un evento además del registro de actividad del servicio Activity. Opcional. Los valores definidos son:

* "notification" - El evento se marca como notificación y será accesible a través del punto final de la API de notificaciones.
* "banner" - El evento se marca como banner.
* "soporte" - El evento se marca como un evento relacionado con el soporte.




| resourceURI | cadena | Falso  a| 
Cadena JSON que contiene el URI asociado al evento. Solo se incluye para eventos relacionados con operaciones de la API. Los valores definidos son:

* Longitud mínima: 3
* Longitud máxima: 4095




| resourceCollectionURL | array[string] | Falso  a| 
Matriz JSON de cadenas JSON, cada una de las cuales indica la URL de acceso completamente expandida del recurso asociado con el evento. Los valores definidos son:

* Longitud mínima: 1
* Longitud máxima: 1023




| resourceMethod | cadena | Falso  a| 
Cadena JSON que contiene el método HTTP invocado. Incluida solo para eventos relacionados con operaciones de API. Los valores definidos son:

* "options" - Operación de la API OPTIONS
* "post" - Operación POST de la API
* "get" - operación GET de la API
* "put" - Operación PUT de la API
* "delete" - operación DELETE de la API




| resourceMethodResult | cadena | Falso  a| 
Cadena JSON que contiene el resultado del método HTTP invocado. Solo se incluye para eventos relacionados con las operaciones de la API. Los valores definidos son:

* La expresión regular definida es: "^[1-5][0-9]{2}$"




| userID | cadena | Falso  a| 
Cadena JSON que contiene el ID de usuario asociado a un evento. Solo se incluye para los eventos asociados a las acciones iniciadas por el usuario. Los valores definidos son:

* Cumple con el esquema del identificador Astra




| accountID | cadena | Falso  a| 
Cadena JSON que contiene el ID de cuenta asociado a un evento. Solo se incluye para eventos asociados a una cuenta. Los valores definidos son:

* Cumple con el esquema del identificador Astra




| data | link:#data["data"] | Falso  a| 
Objeto JSON que contiene información adicional específica del evento.



| metadatos | link:#type_astra_metadata["tipo_astra_metadatos"] | Verdadero  a| 
Metadatos especificados por el cliente y el servicio asociados con el recurso. Los valores definidos son:

* Cumple con el esquema de metadatos Astra
