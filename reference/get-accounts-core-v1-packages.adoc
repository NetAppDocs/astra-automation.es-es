= Lista todos los paquetes
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/packages`#

Devuelve una matriz JSON de todos los recursos coincidentes de la colección. Si se especifican campos individuales en la solicitud, la matriz JSON de elementos contendrá los campos solicitados para cada recurso coincidente en el orden especificado.



== Parámetros

[cols="5*"]
|===
| Nombre | Tipo | En | Obligatorio | Descripción 


| account_id | cadena | ruta | Verdadero  a| 
ID del recurso de cuenta que lo contiene

* ejemplo: {{.Account}}




| filtro | cadena | consulta | Falso  a| 
Indica qué campos deben coincidir para devolver un recurso al listar una colección.

Los operadores admitidos son:

* eq - Igualdad
* lt/gt - Menor que/Mayor que
* lte/gte - Menor o igual que/Mayor o igual que


Ejemplo de URL: "filter=field%20eq%20%27value%27"

Ejemplos OpenAPI: "field eq 'value'"



| OrderBy | cadena | consulta | Falso  a| 
Indica el orden de clasificación de los recursos al listar una colección.

Ejemplos de URL: "orderBy=name", "orderBy=name%20desc"

Ejemplos OpenAPI: "name desc"



| incluya | cadena | consulta | Falso  a| 
Indica qué campos deben devolverse al listar una colección.

Ejemplos de URL: "include=id", "include=id,name"

Ejemplos OpenAPI: "id"

* ejemplo: id,packageName,packageVersion,softwareType




| límite | cadena | consulta | Falso  a| 
Indica el número máximo de recursos a devolver al listar una colección.

Ejemplos de URL: "limit=2"

Ejemplos OpenAPI: "2"

|===

Cadena JSON que describe el nivel de gravedad de este parche o instalación. Los valores definidos son:

* "recommended" - Se recomienda la instalación - Valor por defecto
* "crítico" - Se recomienda encarecidamente la instalación para abordar correcciones de errores críticos y/o parches de seguridad




| packageState | cadena | Verdadero  a| 
Cadena JSON que contiene un valor que indica el estado operativo del paquete. Tras la verificación del paquete y después de la importación, el estado indicará "completo". En caso contrario, el estado indicará "incompleto". Los valores definidos son:

* "verificando" - El paquete está en proceso de validación
* "corrupto" - El paquete está dañado
* "incompleto" - Al paquete le faltan elementos
* "disponible" - Todos los elementos asociados al paquete están disponibles y el paquete está listo para su uso




| packageStateTransitions | array[link:#type_astra_stateTransition["type_astra_stateTransition"]] | Verdadero  a| 
Matriz JSON de objetos JSON, cada uno de los cuales representa el conjunto de transiciones de estado permitidas desde un estado definido dado.
